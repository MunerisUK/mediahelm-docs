Install MediaHelm

MediaHelm is a Docker application. There's no installer to download and no click-to-download link — you run it as a container on your own server or NAS, then open it in a browser. Here's the whole process.

01How Docker install works

MediaHelm is published as a container image to the GitHub Container Registry (GHCR). You don't download a file — Docker pulls the image and runs it. You need Docker on the host first (Docker Desktop on Windows/macOS, or Docker Engine on Linux and most NAS systems). Everything below is a single command you can paste into a terminal.

02The install reference

In Docker terms, this image reference is the "download link" — you give it to Docker, not to a browser:

Image ghcr.io/munerisuk/mediahelm:latest

:latest always resolves to the newest release. To pin a version, swap it for a tag such as ghcr.io/munerisuk/mediahelm:0.2.0.

03Standard install

Publishes the dashboard on port 5005 and keeps your data in a named volume that survives updates and restarts:

bash
$ docker run -d --name mediahelm --restart unless-stopped \
    -p 5005:5005 \
    -v mediahelm-data:/data \
    ghcr.io/munerisuk/mediahelm:latest

Then open http://<host-ip>:5005 (locally, usually http://localhost:5005) and follow the guided setup.

04Host networking (optional)

Use this if you want auto-discovery. For Plex / Emby / Jellyfin to automatically find MediaHelm as an HDHomeRun tuner, it needs a local-network broadcast (UDP 65001) that standard bridge networking blocks. Host networking passes it through. Otherwise the standard install is fine — you'd just add the tuner by URL manually.

bash
$ docker run -d --name mediahelm --restart unless-stopped \
    --network host \
    -v mediahelm-data:/data \
    ghcr.io/munerisuk/mediahelm:latest

With --network host you don't pass -p — MediaHelm uses the host's network directly and still listens on port 5005.

05Docker Compose

Prefer a file you can version and re-run? Save this as docker-compose.yml and run docker compose up -d:

docker-compose.yml
services:
  mediahelm:
    image: ghcr.io/munerisuk/mediahelm:latest
    container_name: mediahelm
    restart: unless-stopped
    ports:
      - "5005:5005"
    volumes:
      - mediahelm-data:/data
    # For HDHomeRun auto-discovery (UDP 65001), remove the
    # "ports" block above and use host networking instead:
    # network_mode: host

volumes:
  mediahelm-data:

Use either the port mapping or host networking — not both.

06Package & source

  • Container package (GHCR)
    The version-browsing page — published tags, pull count and the exact image reference. The closest thing to a web "download page" for the image.
  • Documentation
    Setup, configuration, updates and troubleshooting.
  • Source repository
    The code — for reference and issues. Not a customer download link.

If the package is public, pull it directly — no login. If it's private, sign in to the registry first with docker login ghcr.io (a GitHub username + a token with read:packages).

07"What's the download link?"

There isn't one in the usual sense — MediaHelm installs as a Docker container, not a downloaded installer. The image reference is the install link:

Pullghcr.io/munerisuk/mediahelm:latest

Run the standard install command above, then open http://<your-server>:5005.
